# Test Plan

## Project Overview
This project contains manual QA testing documentation for the DemoBlaze e-commerce demo website.

DemoBlaze is a public demo product store that includes common e-commerce features such as product categories, product pages, cart functionality, sign up, log in, contact form and checkout flow.

## Testing Goal
The goal of this project is to manually test the main user flows of the DemoBlaze website and document test cases, bug reports and testing results.

## Scope
The testing will cover:
- Home page navigation
- Product categories: Phones, Laptops and Monitors
- Product detail pages
- Add to cart functionality
- Cart page
- Place order / checkout flow
- Sign up
- Log in and log out
- Contact form
- Basic UI checks

## Out of Scope
The testing will not cover:
- Real payment processing
- Security testing
- Performance testing
- API testing
- Mobile app testing
- Backend/database testing

## Test Environment
Website: https://www.demoblaze.com  
Browser: Google Chrome  
Device: Desktop / Laptop  
Operating System: Windows  
Testing Type: Manual functional testing

## Test Types
- Functional testing
- UI testing
- Negative testing
- Basic usability testing
- Form validation testing

## Risks and Assumptions
- DemoBlaze is a demo website, so some behavior may not match production-level e-commerce standards.
- Some bugs may be documented as simulated or usability issues for QA practice.
- Testing is limited to a desktop browser environment.

## Deliverables
- Test plan
- Test scenarios
- Manual test cases
- Bug reports
- Screenshots
- Test summary report
